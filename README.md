# AI Browser Assistant - iOS App

An iOS application built with Ionic and Capacitor that provides an AI-powered assistant using LangGraph and OpenAI. This app is a mobile adaptation of the browser extension, using web technologies to create a native iOS experience.

## Features

- **AI-Powered Chat**: Interactive chat interface powered by LangGraph and OpenAI
- **Page Analysis**: Analyze web content and get AI-powered summaries
- **Mermaid Diagrams**: Generate visual diagrams (flowcharts, sequence diagrams, etc.) from content
- **Configuration Management**: Easy setup for API keys and settings
- **Native iOS Experience**: Built with Capacitor for native iOS integration

## Prerequisites

- Node.js (v16 or higher)
- npm or yarn
- Xcode (for iOS development)
- CocoaPods (for iOS dependencies)
- macOS (required for iOS development)

## Installation

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Build the web app:**
   ```bash
   npm run build
   ```

3. **Add iOS platform:**
   ```bash
   npx cap add ios
   ```

4. **Sync Capacitor:**
   ```bash
   npm run sync:ios
   ```

5. **Open in Xcode:**
   ```bash
   npm run ios:open
   ```

## Configuration

1. Open the app in Xcode
2. Navigate to the Configuration page in the app
3. Enter your OpenAI API key
4. Optionally configure LangSmith API key for tracing
5. Save the configuration

### Getting API Keys

- **OpenAI API Key**: Get from [OpenAI Platform](https://platform.openai.com/api-keys)
- **LangSmith API Key** (optional): Get from [LangSmith](https://smith.langchain.com/)

## Development

### Running the Web App (for testing)

```bash
npm run dev
```

This will start a development server at `http://localhost:5173`

### Microsoft Entra (SAML-backed) Authentication for Web

This web app uses Microsoft Entra ID for SSO. Entra can be configured to use SAML with an upstream IdP,
while the app itself uses standard OAuth 2.0 / OIDC for browser-based login.

1. Create an Entra App Registration for a Single-Page Application (SPA).
2. Add redirect URIs (e.g. `http://localhost:5173/`).
3. (Optional) If your org requires SAML, configure Entra as a federated IdP with your SAML provider.
4. Copy `.env.example` to `.env` and set:
   - `VITE_AUTH_ENABLED=true`
   - `VITE_ENTRA_CLIENT_ID=...`
   - `VITE_ENTRA_TENANT_ID=...`
   - `VITE_ENTRA_REDIRECT_URI=...`

After configuring, restart the dev server.

## Key Technologies

- **OpenAI**: LLM provider
- **TypeScript**: Type-safe JavaScript
- **Vite**: Build tool and dev server

## Architecture

### Services

- **StorageService**: Manages app configuration and conversation threads using Capacitor Preferences
- **LangGraphAgentService**: Wraps the LangGraph agent and provides high-level API for chat and analysis

### Pages

- **HomePage**: Main landing page with navigation
- **ConfigPage**: Configuration interface for API keys
- **ChatPage**: Chat interface with message history and AI responses

### Utilities

- **LangGraphAgent**: Core agent implementation using LangGraph
- **MermaidDiagramTool**: Tool for generating Mermaid diagrams
- **Markdown**: Utilities for rendering markdown content

## Differences from Browser Extension

1. **Storage**: Uses Capacitor Preferences instead of Chrome storage
2. **UI**: Uses Ionic components instead of plain HTML/CSS
3. **Navigation**: Uses React Router instead of Chrome extension popup
4. **Platform**: Runs as a native iOS app instead of browser extension
5. **Page Context**: Currently uses mock page data; can be extended to use a webview

## Troubleshooting

### Build Issues

- Make sure you've run `npm install` and `npm run build`
- Ensure Capacitor is synced: `npm run sync:ios`
- Clean build folder in Xcode: Product → Clean Build Folder

### Runtime Issues

- Check console logs in Xcode for errors
- Verify API keys are correctly configured
- Ensure network permissions are set in Info.plist

### iOS Simulator Issues

- Make sure you have Xcode Command Line Tools installed
- Try resetting the simulator: Device → Erase All Content and Settings
